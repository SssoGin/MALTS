# Installing MALTS

MALTS is installed by choosing one or more Agent tool adapters and copying reviewed scaffold files into the matching tool configuration area. Shared MALTS skills are maintained once under the repository root `skills/` directory and are installed into each target tool's local `skills/` directory.

The installer must also make the MALTS runtime discoverable on a new machine. It plans an installed `malts/` runtime copy and a generated `MALTS_BOOT.md` pointer so project initialization can find `README.md`, `skills/`, `runtime/EN/templates`, and `runtime/EN/checklists`.

## Recommended Flow

The recommended first step is dry-run:

```powershell
.\scripts\Install-MALTS.ps1 -Tool Codex
```

If Windows PowerShell blocks script execution, use a process-local execution policy override:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\Install-MALTS.ps1 -Tool Codex
```

For double-click review on Windows, use the wrapper that keeps the console open:

```powershell
.\scripts\Install-MALTS.review.cmd -Tool Codex
```

No files are changed unless `-Apply` is provided:

```powershell
.\scripts\Install-MALTS.ps1 -Tool Codex -Apply
```

## Install Smoke Test

Maintainers can verify a real temporary install without touching the normal tool directories:

```powershell
.\scripts\Test-MALTSInstall.ps1 -Tool AllIncluded
```

The smoke test creates a temporary install target, runs `Install-MALTS.ps1 -Apply -Overwrite` against that temporary target, validates the generated `MALTS_BOOT.md`, runtime templates, checklists, tool scaffold, and `malts/tools/agent_system_lint.py`, then removes the temporary directory unless `-KeepTemp` is provided.

Instruction templates are optional. They can improve MALTS behavior by adding tool-specific reminders for task mode, Grill-Me Preflight, project control, handoff, and verification. Review and merge them with existing user or project instructions before applying.

To skip instruction templates:

```powershell
.\scripts\Install-MALTS.ps1 -Tool ClaudeCode -SkipInstructionTemplate
```

## Defaults

- Shared skills from `skills/` are included by default.
- An installed MALTS runtime copy is included under `<target>\malts\` by default.
- `MALTS_BOOT.md` is generated by default and points to the installed runtime root.
- Runtime templates and checklists are under `runtime/EN` and `runtime/CH`.
- Bilingual documentation sync is disabled by default.
- Codex can install `AGENTS.md` plus Codex-native `.codex` subagent scaffold; Claude Code can install `CLAUDE.md` plus `.claude` agents and commands; OpenCode can install `AGENTS.example.md` as `AGENTS.md` plus `opencode.json` and `.opencode` agents.
- Use `-SkipInstructionTemplate` when the user wants MALTS support files and shared skills without changing Agent instruction files.
- Use `-SkipRuntime` only when another reviewed MALTS runtime root and boot pointer already exist.
- Use `-SkipBoot` only when the tool has another reviewed way to resolve `MALTS_ROOT`.
- Existing files are not overwritten unless the user explicitly allows it.
- The installer prints a plan before writing.

## Skill Install Paths

The repository has one canonical skill source:

```text
skills/
```

The installer copies those skills to the target tool's discovery directory:

```text
Codex      -> <target>\skills\
ClaudeCode -> <target>\skills\
OpenCode   -> <target>\skills\
```

Adapter directories provide tool-specific instruction templates, commands, agents, and configuration. Shared MALTS skills are always installed from the repository root `skills/` directory.

The installed runtime root is:

```text
<target>\malts\
```

The generated boot pointer is:

```text
<target>\MALTS_BOOT.md
```

Agents must resolve `MALTS_ROOT` from `MALTS_BOOT.md` or another configured global boot file before running MALTS project initialization.

## Manual Install

1. Read `README.md`.
2. Choose the target tool adapter.
3. Review the adapter directory under `adapters/`.
4. Copy only the files needed for that tool.
5. Copy root `skills/` into the tool's local `skills/` directory.
6. Keep `runtime/EN/templates`, `runtime/EN/checklists`, and the generated `MALTS_BOOT.md` available to the installed adapter.
7. Run the relevant smoke test or lint command.

See `docs/AGENT_INSTALL.md` for Agent-assisted installation rules.
