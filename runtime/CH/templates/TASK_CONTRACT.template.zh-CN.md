## 权限等级

- Level 0：只读。
- Level 1：可修改指定文件。
- Level 2：可新增文件，但不能删除文件。
- Level 3：重构目录或结构前必须经 Main Controller 批准。
- Level 4：高风险操作，必须用户确认。

## 必须输出

使用 `SUB_AGENT_REPORT` 格式返回结构化报告。

必须包含：

- 运行时 Agent ID，如运行时提供。
- 实际使用模型，如可知；否则说明已记录的模型策略。
- 做了什么。
- 修改了哪些文件，如有。
- 执行了什么验证。
- 哪些内容未验证。
- 风险或阻塞。
- 需要 Main Controller 决策的事项。

Planner 任务还必须包含：

- 建议任务拆分。
- 依赖和优先级。
- 哪些任务已 READY。
- 哪些任务过大、过小或应合并。
- 建议批次大小及理由。

## 升级规则

出现以下情况时必须升级给 Main Controller：

- 任务范围不清楚。
- 需要修改禁止文件。
- 发现严重的范围外问题。
- 验证失败且原因未知。
- 结论与其他 Agent 冲突。
- 需要删除、覆盖或重构文件。

## 安全规则

- 你不是项目里唯一的 Agent。
- 不要回滚或覆盖未知修改。
- 不要自行扩大任务范围。
- 没有验证证据不要声称完成。
- 假设必须标记为假设，不能当作事实。
