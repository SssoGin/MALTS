# MEMORY_WRITE_CHECKLIST

> 写入长期 Skill、`AGENTS.md`、`CLAUDE.md`、等价工具指令入口或全局记忆前使用。

通过本检查清单，表示可以按目标去向写入或提出长期记忆候选；这不是“每一条记忆都必须等用户明确要求才允许写”的总开关。只有当去向会改变持久全局指令或其他高风险行为时，才必须先取得用户确认。

## 事实检查

- [ ] 经验真实发生。
- [ ] 来源明确：项目事件、错误、验证、决策或用户反馈。
- [ ] 事实、判断和假设已区分。
- [ ] 缺失信息已标记，而不是编造。

## 复用价值

- [ ] 类似场景未来可能再次出现。
- [ ] 能减少返工、错误、验证缺口或沟通成本。
- [ ] 收益高于流程成本。
- [ ] 不是一次性局部细节。

## 规则质量

- [ ] 有明确触发条件。
- [ ] 有具体执行动作。
- [ ] 有验证或检查方法。
- [ ] 有适用边界。
- [ ] 说明了要避免的风险或复用的实践。

## 去重

- [ ] 已检查现有 Skills。
- [ ] 如相关，已检查 `AGENTS.md` / `CLAUDE.md` / 等价工具指令入口。
- [ ] 不是用不同说法重复已有规则。
- [ ] 如果已有相似规则，应合并或优化，而不是新增一份。

## 去向判断

| 去向 | 使用场景 |
|---|---|
| 当前复盘 | 只对本项目有用 |
| 项目 Skill | 对本项目类似任务可复用 |
| 全局 Skill | 跨项目可复用 |
| `GLOBAL_MEMORY.md` | 跨项目稳定规则、模式或决策 |
| 工具指令文件候选 | 高频、稳定、跨任务行为，适合写入 `AGENTS.md`、`CLAUDE.md` 或等价工具指令入口 |
| 工具适配文档 | 只适用于 Claude Code、Codex、OpenCode 或其他工具 |
| 不写入 | 太具体、未验证、重复或低价值 |

## 安全

- [ ] 没有写入密钥、令牌、账号或敏感本地细节。
- [ ] 用户敏感信息没有提升到全局记忆。
- [ ] 规则不会导致未来不安全自动化。
- [ ] 高风险规则变更需要用户确认。

## 降级保存

- [ ] 如果目标长期记忆服务、全局 Skill 路径或持久规则目标不可用，候选已保留在 `PROJECT_CONTROL`、工作任务报告或本地复盘记录中，而不是直接丢失。
- [ ] 面向用户的报告区分了“候选建议”“本地降级记录”和“真实长期写入”。
